# WebSWR — Safe Withdrawal Rate & Monte Carlo Simulator

A self-contained, **client-side** retirement calculator in the spirit of cFIREsim.
It runs your portfolio through every historical sequence of returns since **1871**
and through **Monte Carlo** simulation, entirely in your browser — no account, no
server, no tracking, and **zero runtime dependencies**.

## Features

- **Historical engine** — replays every N-year retirement that fits the dataset
  (e.g. a 30-year plan tested starting 1871, 1872, … 1996).
- **Monte Carlo** — three methods, run off the UI thread in a Web Worker:
  *bootstrap* (resample years), *block bootstrap* (resample runs of consecutive
  years), and *parametric* (normal distribution fit to history).
- **Withdrawal strategies** — Constant dollar (the “4% rule”), Percentage of
  portfolio, Guyton-Klinger guardrails, and VPW (variable percentage).
- **Asset classes** — stocks, 10-yr Treasuries, cash (T-bills), gold, plus
  corporate bonds, real estate and US small-cap. Annual rebalancing.
- **Cash flows** — Social Security / pensions and one-off or recurring
  expenses/income, each with an optional inflation (COLA) toggle.
- **Inflation** — historical CPI or a fixed rate. Fees and a simplified tax rate.
- **Max-spending solver** — finds the highest constant spending that meets a
  target success rate, against **history or Monte Carlo**.
- **Real or nominal** balance chart, ending-value distribution, and the
  worst / median / best cycles highlighted.
- **"Rich, broke, or dead?"** — overlays SSA life-table survival odds on your
  outcomes, so you can see whether you're more likely to outlive your money or to
  pass away with a surplus.
- **Loan calculator** — a separate tab amortizes any fixed-rate loan (mortgage,
  car, student loan): payment, total interest, payoff time, balance/interest
  chart, yearly schedule, and how extra payments cut both.
- **Shareable links** — your whole scenario is encoded in the URL hash (no server).
- Light / dark theme, responsive, and an in-app **Help** panel.

## Run it

It’s a static site. The recommended way (enables the Web Worker) is any static
file server:

```sh
cd WebSWR
python3 -m http.server 8000
# open http://localhost:8000
```

You can also just open `index.html` from disk (`file://`). Everything works; the
only difference is that browsers block Web Workers on `file://`, so Monte Carlo
and the solver automatically run on the main thread instead.

There is **no build step** — the scripts are plain classic `<script>` files.

## Host it yourself (Docker + Cloudflare tunnel)

The repo ships the production deployment: a hardened `nginxinc/nginx-unprivileged`
container behind a `cloudflared` tunnel, refreshed daily by `update.sh` (git pull
→ regenerate market/CAPE data → rebuild → restart). Every deploy-specific value
(repo URL, checkout dir, container UIDs/names, URL path) lives in **one file**:

1. `mkdir -p /root/webswr && cd /root/webswr` (any base directory works)
2. Copy [`config.webswr.example`](config.webswr.example) to `config.webswr` and edit it
3. Create `.env` with your tunnel secret: `TUNNEL_TOKEN=...` (`chmod 600 .env`) —
   make the tunnel in the Cloudflare Zero Trust dashboard and route your
   hostname/path to `http://<STACK_NAME>:8080`
4. `curl -fsSLO https://raw.githubusercontent.com/jordanp123/Trimor/main/update.sh`
   (or your fork) and run `bash update.sh` — it clones, fetches fresh data,
   builds and starts everything
5. Root crontab for the daily refresh: `0 5 * * * cd /root/webswr/Trimor && bash update.sh`

`config.webswr` is gitignored and sits in the webroot next to `.env`, so the
nightly update can never overwrite your settings. It contains no secrets.

## Tests

All tests run headlessly via macOS **JavaScriptCore** (`osascript`) — no Node, no
browser, no dependencies:

```sh
sh tests/run.sh
```

Three suites run: the engine + Monte Carlo math (validated against known results
like the 4% rule ≈ 95–96% and SAFEMAX ≈ 3.8%), a UI smoke/integration test that
drives the real form-to-render path through a tiny DOM shim, and a **security
suite** that loads the app with a hostile share-link and asserts the input bounds
hold (see `SECURITY.md`).

## How the data is built

`js/market-data.js` (and `data/market-data.json` for auditing) are generated by a
reproducible, **standard-library-only** Python pipeline:

```sh
python3 tools/fetch_data.py            # uses cached downloads if present
python3 tools/fetch_data.py --refresh  # force re-download
python3 tools/fetch_mortality.py       # SSA life table -> js/mortality-data.js
```

Mortality data comes from the **U.S. Social Security Administration** period life
table (public domain). SSA blocks scripted requests, so `fetch_mortality.py`
parses a browser-saved copy at `tools/.cache/ssa_life_table.html` (refresh by
opening the page in a browser and saving it there).

Sources (all public):

- **1928–present** — Aswath Damodaran (NYU Stern) annual total returns for
  stocks, bonds, bills, corporates, real estate, gold and small-cap; inflation
  from **BLS** CPI-U (Dec→Dec), fetched via multpl.com's Shiller series.
- **1871–1927** — reconstructed from **Robert Shiller’s** annual series (S&P price
  + dividend yield → stock total return; the 10-yr yield → a par-bond total
  return; CPI → inflation). This pre-1928 span is an **annual approximation**
  and covers stocks, bonds and inflation only — gold/cash/corporates/real-estate/
  small-cap begin in 1928, so an allocation that uses them limits cycles to 1928+.

The pipeline re-validates the reconstruction against Damodaran on overlapping
years on every run and refuses to emit data if a source’s format drifts, so you
can re-run it and `git diff` the JSON to see exactly what changed.

## Project layout

```
index.html            app shell, strict Content-Security-Policy, Help modal
css/styles.css        all styles (system fonts only — no web fonts)
js/market-data.js     GENERATED: 1871–2025 returns + inflation (self.SWR_DATA)
js/stats.js           RNG, percentiles, covariance, Cholesky
js/core.js            simulation engine, withdrawal strategies, solver
js/montecarlo.js      bootstrap / block / parametric Monte Carlo
js/charts.js          dependency-free <canvas> charts
js/ui.js              form + results controller
js/worker.js          Web Worker (heavy Monte Carlo / solver)
data/market-data.json GENERATED: same data as JSON, for diffing/audit
tools/fetch_data.py   reproducible data pipeline (Python stdlib only)
tests/                headless tests (engine, UI smoke, security)
SECURITY.md           threat model and hardening
```
